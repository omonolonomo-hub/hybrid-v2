🔄 **Phase 5: Engine Integration (NEXT PHASE)**
Critical architectural disconnects identified:

**Phase 5a: Interface Bridge Fixes**
1. Hand Array Discrepancy - UI expects fixed 6-slot array, engine uses dynamic list
2. Physical Shop Index Discrepancy - `buy_card_from_slot()` missing in real engine
3. Shop Lock Stub - Lock functionality not implemented in engine

**Phase 5b: Time-Dilation Disconnect**
- Engine runs all AI turns in single microsecond
- UI expects human to have 30s shopping time
- Need to split `preparation_phase()` into `start_turn()` + `run_ai_turns()`
- Must register "human" strategy to prevent AI from stealing human's turn

**Phase 5c: Turn-Lifecycle Orchestrator**
Hidden mechanics that must be manually triggered:
1. Interest system (`apply_interest`)
2. Evolution & copy-strengthening checks
3. Market pool bleeding (`return_unsold`)

**Phase 5d: Hidden Catastrophes**
1. Spectator Drift - Dead players get sent back to shop
2. Missing Reroll Method - `reroll_market()` doesn't exist in engine
3. Test Determinism Leakage - RNG drift in passive triggers
4. Ghost Stats - UI shows static stats, ignores dynamic buffs

**Phase 5e: Ultimate Disconnects**
1. Bait-and-Switch Matchup - Pairings regenerated during combat
2. Damage Cap Math Mystery - Early game damage limits not shown
3. Zombie Board - Dead cards not cleaned up from UI
4. Blackout of Evolved Cards - Evolved cards not in CardDatabase
5. get_hand Crash - Missing try-except wrapper

**Phase 5f: TDD Architecture Blueprint** - Test strategy defined
**Phase 5g: Execution Plan** - Implementation order planned
