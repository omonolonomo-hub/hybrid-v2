# T1.4 Phase 2 Complete: GameLoopScene Owns CoreGameState

## Objective
Make GameLoopScene the owner of the game state by building the game with selected strategies and updating CoreGameState.

## Implementation

### Modified Files

#### `scenes/game_loop_scene.py`

**Key Changes:**

1. **Constructor** - Stores CoreGameState from SceneManager
```python
def __init__(self, core_game_state, strategies: Optional[List[str]] = None):
    self.core_game_state = core_game_state  # From SceneManager
    self.strategies = strategies or []
    self.game = None  # Will be built in on_enter()
```

2. **on_enter()** - Builds game and updates CoreGameState
```python
def on_enter(self) -> None:
    if self.game is None:  # Only initialize once
        # Build game with strategies
        self.game = build_game(self.strategies)
        
        # Update CoreGameState with new game
        self.core_game_state.game = self.game
        self.core_game_state.locked_coords_per_player = {
            p.pid: set() for p in self.game.players
        }
        self.core_game_state.view_player_index = 0
        self.core_game_state.fast_mode = False
```

3. **draw()** - Displays game state information
   - Shows turn number
   - Shows current player
   - Lists all players with HP/Gold/Status

### Design Decision: Update vs Create

**Why update CoreGameState instead of creating new?**

SceneManager enforces that `new_scene.core_game_state` must be the same instance (by reference) as the one passed to the factory. This prevents state duplication and ensures all scenes share the same CoreGameState.

**Approach:**
- GameLoopScene receives CoreGameState from SceneManager
- Builds new game with `build_game(strategies)`
- Updates the existing CoreGameState's `game` attribute
- Re-initializes CoreGameState fields for the new game

This maintains SceneManager's reference integrity while allowing GameLoopScene to own the game initialization.

## Data Flow

```
LobbyScene selects strategies
    ↓
Transition to "game_loop" with strategies
    ↓
SceneManager creates GameLoopScene
    ↓
GameLoopScene.__init__(core_game_state, strategies)
    ↓
GameLoopScene.on_enter()
    ↓
build_game(strategies) → Game instance
    ↓
core_game_state.game = new_game
    ↓
CoreGameState updated with new game
```

## Key Features

### 1. Single Initialization
- Game built only once (on first `on_enter()`)
- Subsequent `on_enter()` calls skip initialization
- Prevents duplicate game creation

### 2. SceneManager Compatibility
- CoreGameState passed by reference (not copied)
- Same instance shared across all scenes
- Satisfies SceneManager's assertion checks

### 3. State Persistence
- `self.game` persists across scene transitions
- `self.core_game_state` persists across scene transitions
- Only `self.ui_state` is discarded on `on_exit()`

### 4. Debug Logging
- Detailed console output for validation
- Shows player count, strategies, gold, HP
- Confirms initialization status
