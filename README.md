## 🚀 Overview
**Hybrid: Reborn** is a solo-developed, agent-assisted prototype for a hex-grid based autochess simulation. This project serves as a **"Vibecoding" Testbed**—an experiment in rapid development where complex game logic, mathematical balancing, and UI rendering are orchestrated through high-level architectural prompts and AI collaboration.

The engine simulates an 8-player lobby where players draft from a massive **101-card pool** across diverse categories like Mythology, Technology, and Abstract Concepts to survive automated hex-grid combat.

## 🧠 The "Vibecoding" Context
This is not a traditional repository. It is a **living prototype** built using an "Agent-First" philosophy:
- **Strict GDD Enforcement:** The logic is governed by a machine-readable Game Design Document (GDD).
- **Architecture over Spaghetti:** Transitioning from a monolithic legacy system to a modular, scene-based architecture.
- **Rapid Iteration:** Developed using high-speed AI-assisted coding to test complex interactions between 101 unique passives.

## 🧬 Key Features
- **101 Unique Cards:** Each with distinct stats (Meaning, Gravity, Trace, etc.) and passive abilities.
- **Hex-Grid Combat:** Complex positioning logic including neighboring synergies and field effects.
- **Hybrid Categories:** Cards bridge the gap between ancient Mythology, Modern World structures, and Sci-Fi elements.
- **AI Strategy Simulation:** Built-in AI agents (Warrior, Economist, Evolver) to test meta-stability.
- **Cyber-Synthwave Aesthetic:** A neon-driven UI designed for a professional simulation feel.

## 📂 Repository Structure (WIP)
- `engine_core/`: The "Brain" – Simulation logic, combat resolution, and player state.
- `ui/`: The "Face" – Pygame-CE rendering, lobby scenes, and card animations.
- `assets/data/`: The "Soul" – `cards.json` and passive definitions.
- `tests/`: Automated simulations to track KPI and game balance.
